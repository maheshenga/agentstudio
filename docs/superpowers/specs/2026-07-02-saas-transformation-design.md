# SaaS 化改造设计

日期：2026-07-02

## 目标

将当前的多租户后台系统改造成完整商业 SaaS 平台。

目标系统继续保留当前 Vue 3 + Vite 前端和 NestJS 单体后端，但新增 SaaS 产品能力：

- 用户自助注册并开通租户。
- 平台管理员手动创建租户和邀请用户。
- 默认免费版，同时赠送高级功能试用。
- 付费套餐、订单、订阅、资源包和支付宝支付。
- 用量统计和额度限制。
- 同一个前端入口内区分平台视角和租户视角。

## 已确认的产品决策

- 注册模式：支持用户自助开通租户，也支持平台管理员创建租户。
- 默认订阅：注册后默认免费版，同时赠送高级功能试用。
- 租户默认角色：`owner`、`admin`、`member`。
- 前端入口：一个前端应用，根据身份和菜单区分平台视角/租户视角。
- 商业模型：套餐订阅 + 超额资源包。
- 支付架构：多支付渠道适配器，第一版真实实现支付宝。
- 额度维度：用户数、存储空间、AI 调用次数、知识库文档数、Token 用量。

## 当前项目基础

当前系统已有一些可复用基础：

- `sa_system_tenant`：租户表。
- `sa_system_user`：用户表。
- `sa_system_user_tenant`：用户和租户关系表。
- `sa_system_role`、`sa_system_menu` 以及角色菜单关系：RBAC 权限基础。
- NestJS 后端已有全局认证、权限、多租户拦截器。
- Vue 前端已经支持后端菜单驱动的动态路由。
- AI 和 RAG 模块可以接入统一额度控制。

当前缺口：

- 前端注册页是模拟注册，没有真实调用后端注册接口。
- 后端注册只创建用户，不会创建租户、订阅、默认角色和租户成员关系。
- 没有完整平台计费模型。
- 平台管理员、租户管理员、订阅和额度边界还不明确。
- 订单、支付、订阅、资源包、发票模型尚不存在。

## 总体架构

保留一个后端和一个前端，但在逻辑上分成两个视角。

平台视角：

- SaaS 控制台。
- 租户管理。
- 套餐管理。
- 订单管理。
- 支付流水。
- 订阅管理。
- 资源包管理。
- 用量统计。
- 发票管理。
- 平台设置。

租户视角：

- 工作台。
- 成员管理。
- 角色权限。
- 套餐与订阅。
- 资源包。
- 用量中心。
- AI 对话。
- RAG 知识库。
- 现有业务模块。

同一个账号如果同时拥有平台权限和租户权限，可以在顶部切换视角。切换视角后重新拉取菜单、首页路径和当前租户上下文。

## 数据模型

新增 SaaS 商业表，不直接把商业逻辑塞进现有系统表。

### 套餐

`saas_plan`

- 套餐表。
- 定义免费版、专业版、企业版。
- 存储套餐名称、编码、计费周期、状态、排序和描述。

`saas_plan_feature`

- 套餐功能开关表。
- 示例：是否允许 AI、RAG、高级报表、成员管理。

`saas_plan_quota`

- 套餐额度表。
- 额度类型：用户数、存储、AI 调用次数、知识库文档数、Token 用量。

### 订阅和试用

`saas_subscription`

- 租户订阅表。
- 一个租户同一时间只有一个当前有效订阅。
- 存储租户、套餐、计费周期、状态、开始时间、结束时间、续费策略和取消状态。

订阅状态：

- `trialing`：试用中。
- `active`：正常订阅。
- `past_due`：待付款或续费失败。
- `expired`：已过期。
- `cancelled`：已取消。
- `frozen`：已冻结。

`saas_trial`

- 试用权益表。
- 存储试用开始时间、结束时间和试用功能范围。

### 订单和支付

`saas_order`

- 订单表。
- 存储套餐订单、续费订单、升级订单和资源包订单。
- 存储订单号、租户、用户、金额、状态、订单类型和来源对象。

`saas_payment`

- 支付流水表。
- 存储支付渠道、支付宝交易号、支付状态、请求参数、回调内容、支付时间和金额。

支付渠道统一接口：

- `createPayment(order)`
- `verifyNotify(payload)`
- `queryPayment(orderNo)`
- `closePayment(orderNo)`
- `refundPayment(orderNo)`

第一版实现：

- `AlipayPaymentProvider`

后续可扩展：

- `WechatPaymentProvider`
- `StripePaymentProvider`
- `ManualPaymentProvider`

### 资源包和用量

`saas_resource_pack`

- 资源包定义表。
- 示例：AI 调用包、Token 包、存储包、文档包、用户席位包。

`saas_tenant_resource`

- 租户资源余额表。
- 存储套餐基础额度、资源包叠加额度、已用量和剩余额度。

`saas_usage_record`

- 用量流水表。
- 示例：成员新增、文件上传、AI 调用完成、Token 消耗、文档索引。

`saas_invoice`

- 发票/账单申请表。
- 第一版可以先由平台管理员手动处理。

### 现有表增强

`sa_system_tenant`

- 增加 SaaS 状态字段：当前套餐、订阅到期时间、欠费状态、冻结状态。

`sa_system_user_tenant`

- 继续表示用户属于哪个租户。
- `owner/admin/member` 可以通过默认角色表示，必要时再增加显式成员身份字段。

`sa_system_role` 和 `sa_system_menu`

- 继续作为 RBAC 基础。
- 新增 SaaS 平台菜单和租户账单菜单。

## 注册和租户开通

自助注册流程：

1. 用户填写邮箱或手机号、密码、姓名、组织名称、可选行业/团队规模、验证码。
2. 创建 `sa_system_user`。
3. 创建 `sa_system_tenant`。
4. 创建 `sa_system_user_tenant` 成员关系。
5. 设置注册用户为租户所有者。
6. 创建默认角色：`owner`、`admin`、`member`。
7. 给注册用户分配 `owner` 角色。
8. 创建免费版订阅。
9. 创建高级功能试用权益。
10. 初始化租户资源额度。
11. 自动登录并进入租户工作台。

平台管理员创建租户流程：

1. 平台管理员创建租户。
2. 平台管理员选择套餐和是否赠送试用。
3. 平台管理员创建租户所有者账号，或邀请已有用户。
4. 系统初始化默认角色、订阅和额度。
5. 租户所有者登录后管理租户。

成员邀请流程：

1. 租户所有者或管理员输入邮箱/手机号。
2. 选择角色：`admin` 或 `member`。
3. 系统生成邀请令牌。
4. 已有用户接受邀请后加入租户。
5. 新用户先注册，再加入租户。

## 登录和视角选择

登录流程：

1. 用户输入账号。
2. 系统加载用户可访问的租户。
3. 如果只有一个租户，则自动选择。
4. 如果有多个租户，则用户选择一个。
5. 用户输入密码和验证码。
6. 后端创建带租户上下文的会话。
7. 前端拉取用户信息、字典和菜单。
8. 前端根据后端菜单注册动态路由。

如果用户拥有平台权限，显示平台视角。如果用户同时属于租户，则允许在顶部切换平台视角和租户视角。

## 套餐、订单和支付宝

内置套餐：

- 免费版：永久可用，低额度，基础功能，赠送高级试用。
- 专业版：支持月付/年付，额度更高，可以购买资源包。
- 企业版：额度更高，可以由平台管理员手动开通。

购买套餐流程：

1. 租户所有者选择套餐和计费周期。
2. 后端创建 `saas_order`。
3. 后端通过支付适配器创建支付宝支付单。
4. 前端跳转支付宝收银台或展示二维码。
5. 支付宝发送异步回调。
6. 后端校验签名。
7. 后端更新支付状态。
8. 后端更新订单状态。
9. 后端创建或更新订阅。
10. 后端刷新租户额度。
11. 后端写操作日志。

购买资源包流程：

1. 租户所有者选择资源包。
2. 后端创建资源包订单。
3. 用户使用支付宝支付。
4. 支付回调成功后增加租户资源余额。

升级和续费规则：

- 续费立即延长当前订阅。
- 升级立即生效。
- 降级下个周期生效。
- 退款第一版由平台后台手动处理。

## 额度和用量控制

额度维度：

- 用户数。
- 存储空间。
- AI 调用次数。
- 知识库文档数。
- Token 用量。

统一服务：

`SaasQuotaService`

- `checkQuota()`
- `consumeUsage()`
- `releaseUsage()`
- `getTenantUsageSummary()`

控制点：

- 邀请成员和添加成员时检查用户数额度。
- 文件上传时检查存储额度并记录文件大小。
- 文件删除后释放存储，或标记为延迟释放。
- AI 请求前检查 AI 调用次数和 Token 额度。
- AI 完成后记录 prompt tokens、completion tokens、total tokens。
- 模型不返回 token 用量时，按文本长度估算。
- 创建知识库文档时检查文档数额度。
- 删除知识库文档后释放文档额度。

扣减顺序：

1. 校验订阅状态。
2. 检查套餐基础额度。
3. 检查资源包额度。
4. 执行业务功能。
5. 记录用量流水。
6. 更新租户资源汇总。

超额策略：

- 用户数超额：禁止邀请和新增成员。
- 存储超额：禁止上传。
- AI 调用次数超额：禁止发起 AI 请求。
- Token 超额：禁止发起 AI 请求，并提示购买 Token 包。
- 文档数超额：禁止创建知识库文档。
- 试用结束：关闭试用专属功能，保留免费版能力。
- 订阅过期：降级到免费版或冻结付费功能。

## 权限和菜单

身份分层：

- 平台超级管理员。
- 平台运营人员。
- 租户所有者。
- 租户管理员。
- 租户普通成员。

平台权限示例：

- `saas:tenant:index`
- `saas:tenant:update`
- `saas:plan:index`
- `saas:plan:update`
- `saas:order:index`
- `saas:payment:index`
- `saas:subscription:update`
- `saas:usage:index`
- `saas:invoice:update`

租户权限示例：

- `tenant:billing:view`
- `tenant:billing:upgrade`
- `tenant:member:index`
- `tenant:member:invite`
- `tenant:quota:view`
- `tenant:resource:buy`

关键原则：

- 平台接口不能误用当前租户过滤。
- 租户接口必须要求租户上下文，并且只能使用当前 `tenantId`。

## 分期计划

### 第 1 期：SaaS 基座

目标：先让系统具备 SaaS 产品形态，但不接真实支付。

范围：

- 自助注册创建租户。
- 平台管理员创建租户。
- 默认角色：`owner`、`admin`、`member`。
- 免费套餐。
- 高级功能试用。
- 基础订阅记录。
- 额度初始化。
- 平台视角和租户视角分离。
- 基础套餐页。
- 基础用量中心。

不包含：

- 真实支付宝支付。
- 资源包购买。
- 自动发票处理。
- 复杂升级/降级计费。

### 第 2 期：套餐、订单和支付宝

范围：

- 套餐管理。
- 订单管理。
- 支付宝支付适配器。
- 支付宝异步回调。
- 订阅开通、续费、升级。
- 支付流水。
- 手动退款记录。

### 第 3 期：资源包和精细用量

范围：

- 资源包购买。
- 用户数、存储、AI 调用次数、文档数、Token 用量统计。
- 额度扣减。
- 超额限制。
- 租户用量看板。
- 平台用量统计。

### 第 4 期：商业运营增强

范围：

- 发票申请。
- 企业版手动开通。
- 支付失败提醒。
- 订阅到期提醒。
- 账单报表。
- 收入统计。
- 租户健康度。

## 测试策略

后端：

- 租户开通单元测试。
- 额度检查和扣减单元测试。
- 订单状态流转单元测试。
- 支付渠道签名校验单元测试。
- 注册到登录集成测试。
- 支付宝回调幂等性集成测试。

前端：

- 使用 `vue-tsc` 做类型检查。
- 注册、套餐购买、资源包购买表单校验。
- 平台菜单和租户菜单的权限可见性校验。

数据库：

- 新增 SaaS 表迁移测试。
- 默认套餐、角色、菜单、额度种子数据测试。

运行检查：

- 健康检查保持正常。
- 现有管理员登录继续可用。
- 现有租户数据继续可访问。
- AI/RAG 接入额度后保持原有流程可用。

## 实施注意事项

- 先完成第 1 期 SaaS 基座，再接真实支付宝支付。
- 现有注册页要从模拟注册改为真实 API 调用。
- 现有登录页的租户选择器可以复用。
- 现有租户管理模块可以扩展，但套餐、订单、支付、订阅应放到独立 SaaS 模块。
- 继续复用现有 RBAC，不重写权限系统。
- AI 和 RAG 模块要在明确的使用边界调用 `SaasQuotaService`。
